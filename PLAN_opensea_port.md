## 0. 대전제

- **신규 구축이 아니라 이식이다.** 기능은 이미 React 앱과 test_dashboard 안에 존재한다. 새로 발명하지 않고, 있는 걸 가져와 OpenSea UI/UX에 맞춘다.
- OpenSea 레퍼런스(좌측 nav · 우측 날개 · 상단 브레드크럼)는 골격으로 유지하고, 그 안에 기능을 채운다.
- 게임플레이 DNA를 다시 뒤로 밀지 않는다. 지난번처럼 타이포에 집중하다 DNA를 죽이는 실수를 반복하지 않는다.
- 우측 날개 = **현황(status) 패널**. 보고 있는 뷰나 선택한 항목의 요약 정보를 띄운다.
- 무의미한 영어("Game Designer Portfolio", "Resume & Cover Letter" 등)와 군더더기 긴 설명은 지운다.

확인된 현재 상태:
- 좌측 nav 5개 뷰: 홈 / 이력 및 자기소개 / 포트폴리오 / 플레이 이력 / 애니메이션 이력
- 플레이·애니메이션: ~90% 완성 / 홈·이력·포트폴리오: 미완(이식 대상)
- 포트폴리오 데이터 `PORTFOLIO_ITEMS`, 게임 DNA `GAME_DNA_CHART_DATA` 모두 이미 파일에 임베드됨

## Phase 1 — 플레이 / 애니메이션 폴리싱 (가벼움, 검증 위주)

지난번 복구분이 살아있는지 검증하고, 타이포만 애니메이션 기준으로 맞춘다. 새 기능 추가 없음.

1. **게임플레이 DNA 검증**
   - 8각형 레이더(RPG·어드벤처·퍼즐·액션·전략·시뮬레이션·슈팅·리듬) 정상 렌더 확인
   - 꼭짓점 hover/클릭 인터랙션 확인. 점수 숫자 표기(예: RPG → 99)가 hover 시 안 뜨면 추가한다. 데이터(`GAME_DNA_CHART_DATA[].score`)는 이미 있으므로 표시만 붙이면 됨
   - 하이라이트 4종(메이플스토리 / DARK SOULS™ III / 무기미도 / osu!) 썸네일이 참조하는 `GAME_HISTORY_ITEMS[].image` 경로가 실제로 로드되는지 검증. 깨지면 경로만 수정
2. **타이포그래피 통일** — 애니메이션 기준
   - `#view-play .section-title`(+ 필요 시 base `.section-title`)에 애니메이션과 동일한 `font-size:23px; font-weight:800` 적용 → "전체 게임"을 "전체 애니메이션"과 같은 크기로
   - 섹션 헤더 구조(`section-title-wrap / -copy / -row`)도 애니메이션 쪽 정렬에 맞춤
3. 두 뷰의 우측 날개(최애 게임 / 최애 애니메이션)는 현황 패널 컨셉에 이미 부합 → 유지

## Phase 2 — 포트폴리오 디테일 팝업 (Steam 게임 상점 페이지 컨셉)

핵심 미구현부. 현재 카드 클릭 시 `// Can expand to a modal later`만 있고 디테일이 없다.
방향: **OpenSea 톤으로 재해석한 Steam 게임 상점 페이지.** React의 윈도우 브라우저 크롬(주소창·탭바·창 버튼)은 버리고, Steam 스토어 느낌(히어로 배너 + 좌측 영상 + 탭)으로 OpenSea 다크 톤에 통일한다.

1. **클릭 → 팝업 동작**
   - 포트폴리오 카드 클릭 시 **가운데 콘텐츠 영역**에 디테일이 크게 뜬다
   - 좌측 nav · 우측 날개 · 상단 브레드크럼은 그대로 남는다 (전체 화면 오버레이 아님)
   - 상단 브레드크럼은 `포트폴리오 / [작품명]`, 닫기(X)로 그리드 복귀
2. **Steam 상점 스타일 본문** (React `ProjectDetail` 개요 탭이 이미 이 스타일 → 이식)
   - 상단 히어로 배너(키아트 + 역할 배지 + 제목)
   - 좌측: 플레이 영상(YouTube) 또는 키아트, 그 아래 짧은 설명
   - 탭: 개요 / 기획서 / 시나리오 기획서 / 프로토타입 / 플레이 영상 / 링크 / 시뮬레이터 / 간트차트 — 데이터에 해당 필드가 있을 때만 노출(React의 `show` 조건 그대로)
   - 다크 톤은 OpenSea 팔레트로 통일(기존 Steam의 #1B2838 계열을 OpenSea 변수로 매핑)
3. **우측 날개 = 선택 포트폴리오 현황**
   - React 개요 탭 우측 패널(팀 규모 / 주요 역할 / 주요 성과 / 이 게임의 인기 태그)을 **우측 날개로 이동**
   - 팝업 열림 → 날개가 해당 작품 정보로 갱신, 닫힘 → 기존 "프로젝트 요약"으로 복귀
4. **정리**
   - line 4768 `기획 문서를 실제 플레이 경험으로 연결한 프로젝트` (포트폴리오 헤더) **삭제**
   - 포트폴리오 그리드는 `PORTFOLIO_ITEMS` 그대로 사용, 카드에 onclick 바인딩

## Phase 3 — 이력 및 자기소개 분리

한 nav 항목을 유지하되 안에서 두 페이지를 버튼으로 전환한다.

1. **세그먼트 토글** `[ 이력 | 자기소개 ]` 를 뷰 상단에 배치, 클릭 시 두 영역 전환
2. **이력 페이지**: React `Resume`(ResumeHeader / Experience / Education / Tools) 내용 이식
3. **자기소개 페이지**: React `CoverLetter` / `About` 내용 이식
4. 영어 kicker `Resume & Cover Letter`(line 2126) 제거 또는 한글화
5. 우측 날개 = 현재 페이지(이력/자소서) 현황 요약
